++ System run levels
--- who -r (Permite mostar en que modo nos encontramos)
--- Main run levels:
----- init 0 (Apaga el equipo)
----- init 1 (Modo un sólo usuario)
----- init 6 (Reinicia el equipo)
--- Other run levels:
----- init 2 (Modo multiusuario sin red)
----- init 3 (Modo multiusuario sin gui y con red)
----- init 5 (Modo multiusuario con gui y con red)

++ Cambiar permisos al crear archivos nuevos
--- umask u+rw,g+r,g-w,o-rwx 
---- Usuario actual: Permisos de lectura y escirtura
---- Grupo: Permisos de lectura
---- Otros: Sin ningún permiso
---- Nota: esta asignación se perderá al cerrar sesión o reiniciar el equipo.
---- Nota: para hacer permanente está asignación debemos editar el archivo .bashrc, al final de este podemos agregar el comando umask correspondiente

++ Colores en el sistema de archivos
--- Azul = Directorio
--- Verde = Ejecutable 
--- Celeste = Enlace simbólico
--- Amarrillo con fondo negro = dispositivo
--- Rosado = imagen
--- Rojo = archivo tar
--- Rojo con fondo negro = Enlace simbólico roto
---- Nota: estos colores pueden cambiarse si se desea.
---- Nota: no todas las distribuciones tienen este mismo esquema de colores

++ Comandos útiles para unidades de almacenamiento
--- df -h (Muestra las unidades de almacenamiento en formato legible para humanos)
--- df -T (Muestra los formatos asociados a las unidades de almacenamiento)
--- df -h | grep -v tmpfs (Muestra las unidades de almacenamiento y excluye aquellas con etiquetas de temporal, las cuales en realidad no son unidades de almacenamiento)
--- fdisk -l (Permite ver las unidades de almacenamiento y particiones)
--- fsck /dev/<nombre_almacenamiento> (Corrige a nivel de bloque los errores en la unidad de almacenamiento o partición)
---- Nota: para que el comando funcione la unidad de almacenamiento o partición NO debe estar montada)
---- Ejemplo de uso: fsck -y /dev/sda1 
---- Ejemplo de uso (particion xfs): fsck.xfs /dev/sda1
------ Luego ejecutar: xfs_repair /dev/sda1
